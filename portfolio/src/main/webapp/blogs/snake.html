Welcome to my second post! In this blog, I present a javascript snake game that I made, with inspiration from the tutorial here. I modified the tutorial to have classes instead of functions to get a better feel for object oriented programming in javascript.

Here's an intersesting bug that I found in the tutorial: When pressing mutliple keys at the same time within the same animation interval (in this case 1/10 of a second), the snake bugs out and hits itself when making a tight turn. For example, if we send two commands such as up and left, and the snake is currently moving right, because they occur in the same time interval, the snake never physically moves up. Both updates happen right after each other, but only the left turn is actually animated. Thus the snake hits itself and the game prematurely ends. There is a simple fix: Create a queue of direction updates so we can ensure each change in direction gets animated one by one.
